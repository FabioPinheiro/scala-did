Getting Started with Scala-did

To install ScalaDID in a scala project

 libraryDependencies += "app.fmgp" %% "did" % 0.1.0-M25+231-b50bbd5e-SNAPSHOT // for DID and DID Comm
 libraryDependencies += "app.fmgp" %% "did-imp" % 0.1.0-M25+231-b50bbd5e-SNAPSHOT // for crypto implementation
 libraryDependencies += "app.fmgp" %% "did-framework" % 0.1.0-M25+231-b50bbd5e-SNAPSHOT //for utils
 libraryDependencies += "app.fmgp" %% "did-method-peer" % 0.1.0-M25+231-b50bbd5e-SNAPSHOT // for resolver of the did method `peer`
 libraryDependencies += "app.fmgp" %% "did-method-web" % 0.1.0-M25+231-b50bbd5e-SNAPSHOT // for resolver the did method `web`
 libraryDependencies += "app.fmgp" %% "did-uniresolver" % 0.1.0-M25+231-b50bbd5e-SNAPSHOT // for calling the resolver uniresolver

In a crossProject for the JSPlatform and JVMPlatform this should be %%% instead of %%

Modules

Let got module by module

  • did -

  • did-imp

  • did-framework

  • did-peer & did-web is our implementation of of the did:peer and did:web methods.

  • did-uniresolver is an intregation with the uniresolver. So you can use the uniresolver to resolver the did methods. You can use the publicly available uniresolver or a custom one.

  • did-example is just a list DID knowd identities, Like mediator and other agents. Also, contends some utilities.

  • multiformats is our implementation of Multiformats used by did-peer
