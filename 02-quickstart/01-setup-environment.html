Setup environment

Setup an environment for quick start

The Library is published to through sonatype maven.

scala-cli on docker

Start an isolated environment to experiment with some code samples (3/5 mins)

docker run --rm -it --entrypoint /bin/sh virtuslab/scala-cli

scala-cli repl \
  --dependency app.fmgp::did::0.1.0-M24+8-61c431b6-SNAPSHOT \
  --dependency app.fmgp::did-imp::0.1.0-M24+8-61c431b6-SNAPSHOT \
  --dependency app.fmgp::did-method-peer::0.1.0-M24+8-61c431b6-SNAPSHOT \
  --repo https://oss.sonatype.org/content/repositories/releases

# For snapshots use
# --repo https://oss.sonatype.org/content/repositories/snapshots
# For releases use
# --repo https://oss.sonatype.org/content/repositories/releases

SBT setup

To install the library on sbt, you can use the following lines to your build.sbt:

 libraryDependencies += "app.fmgp" %% "did" % 0.1.0-M24+8-61c431b6-SNAPSHOT
 libraryDependencies += "app.fmgp" %% "did-imp" % 0.1.0-M24+8-61c431b6-SNAPSHOT // for the DIDComm implementation
 libraryDependencies += "app.fmgp" %% "did-resolver-peer" % 0.1.0-M24+8-61c431b6-SNAPSHOT // for hash utils

In a crossProject for the JSPlatform and JVMPlatform this shoud use this instead:

 libraryDependencies += "app.fmgp" %%% "did" % 0.1.0-M24+8-61c431b6-SNAPSHOT
 libraryDependencies += "app.fmgp" %%% "did-imp" % 0.1.0-M24+8-61c431b6-SNAPSHOT // for the DIDComm implementation
 libraryDependencies += "app.fmgp" %%% "did-resolver-peer" % 0.1.0-M24+8-61c431b6-SNAPSHOT // for hash utils
